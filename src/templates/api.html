  • 用户注册

    接口地址：/user/register

    请求方式：POST

    请求参数:空
    参数名 说明 是否必填
    account 用户注册号（手机号码） 是
    password 用户密码 是
    pic 用户头像（转为base64格式） 是
    nickname 用户昵称 是
    sex 用户性别(0=男；1=女) 是
    province 所在省 是
    city 所在市 是

    返回格式：JSON

    返回列表
    code http状态码，只有200表示结果正确
    data 返回的导航数组列表
      -id 导航id号
      -name 导航显示名称
    msg 服务器返回消息

    返回示例

                            {
                              "code": 200,
                              "data": [
                                    {
                                          "id": 101,
                                          "name": "推荐"
                                    },
                                    {
                                          "id": 102,
                                          "name": "母婴"
                                    },
                                    {
                                          "id": 103,
                                          "name": "数码"
                                    },
                                    {
                                          "id": 106,
                                          "name": "test"
                                    }
                              ],
                              "msg": "ok"
                        }

  • 主办方活动

    接口地址：/activity/sponsor

    请求方式：GET

    请求参数
    参数名 说明 是否必填
    label 标签名 否
    attribute 属性(1=最新；2=热门；3=推荐；4=免费；5=附近；6=精选) 否，没有该属性时，代表是活动首页
    date 日期(转为时间戳) 否
    city_id 城市id 是
    page_size 页面大小 是
    page_index 页码 是
    longitude 经度（当属性为附近时，必填） 否
    latitude 纬度（当属性为附近时，必填） 否

    返回格式：JSON

    返回示例

                            {
                          "data": [
                            {
                              "address": "zhong yue",   //活动地址
                              "attend_count": 0,        //活动参加人数
                              "begin_time": 1454571510, //活动开始时间
                              "cast_much": "",          //活动消费
                              "name": "",               //主办方名字
                              "pic": "1.jpg"            //主办方头像
                              "id": "56b05cf794a7c0a790bec7da",   //活动id
                              "is_collection": false,             //是否收藏  true=已经收藏   false=未收藏
                              "labels": [                         //活动标签
                                "\u65c5\u6e38",
                                "\u6c99\u53d1"
                              ],
                              "poster": "img/1.png",             //活动海报
                              "recommend_count": 0,              //活动推荐人数
                              "title": "qu da pai"               //活动标题
                            }
                          ],
                          "msg": "\u83b7\u53d6\u6570\u636e\u6210\u529f",
                          "status": true
                        }

  • 获取某一导航下条目列表

    接口地址：/v1.0/Channel/<int:channelId>/Item

    请求方式：GET

    请求参数:
    offset 请求的数据偏移量（为空则默认为0）
    page_size 请求的数据页面大小（为空则默认为20）

    返回格式：JSON

    返回列表
    code http状态码，只有200表示结果正确
    data 返回的类目数组列表
      -id 类目id号
      -title 类目标题
      -cover_image_url 类目封面显示图片地址
      -publish_time 类目发布时间（timestamp）
      -type 类目种类（根据不同的种类，在客户端的点击显示格式也不一样，目前只有post）
    msg 服务器返回消息

    返回示例

                            {
                          "code": 200,
                          "data": [
                            {
                              "cover_image_url": "http://img_url.jpg",
                              "id": 2,
                              "publish_time": 1443575543.0,
                              "title": "标题1",
                              "type": "post"
                            },
                            {
                              "cover_image_url": "http://img_url.jpg",
                              "id": 3,
                              "publish_time": 1443575543.0,
                              "title": "标题2",
                              "type": "post"
                            }
                          ],
                          "msg": "ok"
                        }
